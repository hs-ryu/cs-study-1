## 1. 데이터베이스 관리 시스템의 등장 배경

- 과거에는 데이터를 관리하기 위해 파일 시스템이라는 소프르웨어를 이용하였음. 데이터를 파일로 관리할 수 있도록 파일을 생성,삭제,수정,검색하는 기능을 제동하며, 운영체제와 함께 설치됨. 파일 시스템은 별도의 구매 비용이 들지 않는다는 장점이 있지만, 응용프로그램마다 파일을 따로 유지하는 특징 때문에 단점들이 존재함. 파일 시스템의 단점은 다음과 같다.

 1. 같은 내용의 데이터가 여러 파일에 중복 저장됨.
 2. 응용 프로그램이 데이터 파일에 종속적임.
 3. 데이터 파일에 대한 동시 공유, 보안, 회복 기능이 부족함.
 4. 응용 프로그램을 개발하기 쉽지 않음.





## 2. 데이터베이스 관리 시스템의 정의

- 파일 시스템이 지닌 문제를 해결하는 1차 방안은 데이터를 통합하여 저장하는 것임.
- 하지만 통합 저장된 데이터를 관리하고 모든 응용 프로그램이 공통으로 요구하는 데이터에 대한 기본 처리를 담당하면서 동시 공유, 보안, 회복 등의 복잡한 기능을 제공해주는 새로운 무엇이 필요함. 그 대안으로 제시된 것이 데이터베이스 관리 시스템임.
- 데이터베이스 관리 시스템 : 파일 시스템의 데이터 중복과 데이터 종속 문제를 해결하기 위해 제시된 소프트웨어. 응용 프로그램을 대신하여 데이터베이스에 들어있는 데이터를 삽입,삭제,수정,검색하고 모든 응용프로그램이 데이터베이스를 공유할 수 있게 함.
- 데이터베이스 관리 시스템이 제공하는 주요 기능
  - 정의 기능 (데이터베이스 구조를 정의하거나 수정할 수 있음)
    - 데이터베이스 관리 시스템은 조직에 필요한 데이터를 저장하기 적합한 데이터베이스 구조를 정의하거나, 이미 정의된 구조를 수정할 수 있음.
  - 조작 기능 (데이터를 삽입,삭제,수정,검색하는 연산을 할 수 있음)
    - 데이터베이스 관리 시스템은 데이터베이스에 저장된 데이터에 접근하여 사용할 수 있는 기능을 제공함. 즉, 사용자 요구에 따라 데이터를 삽입,삭제,수정,검색하는 연산을 효율적으로 처리함.
  - 제어 기능 (데이터를 항상 정확하고 안전하게 유지할 수 있음)
    - 데이터를 여러 사용자가 공유해도 항상 정확하고 안전하게 유지하는 기능을 제공함. 데이터를 삽입,삭제,수정하는 연산을 한 후에도 내용이 일관되면서 무결성을 유지하게 하고, 장애가 발생해도 회복이 가능하도록 제어함. 또한 권한이 있는 사용자에게만 데이터 접근을 허용하여 보안이 유지되도록 제어함. 특히, 여러 사용자가 데이터베이스에 동시에 접근하여 데이터를 처리할 수 있도록 제어함.



## 3. 데이터베이스 관리 시스템의 장단점

#### 데이터베이스 관리 시스템의 장점

> 파일 시스템과 비교해 다음과 같은 장점이 존재함.

1. 데이터 중복을 통제할 수 있음.

- 파일 시스템은 응용 프로그램마다 별도의 파일을 유지하기 때문에 같은 내용의 데이터를 여러 파일에 중복 저장될 수 있음. 데이터가 중복되면 저장 공간이 낭비되고 데이터를 저장하고 수정하는 비용도 증가함.
- 데이터베이스 관리 시스템은 데이터베이스에 데이터를 통합하여 관리하므로 데이터 중복문제 해결 가능함. 일관성도 유지할 수 있음.

2. 데이터 독립성이 확보됨.

- 파일 시스템에는 파일 구조가 바뀌면 응용 프로그램도 함께 수정해야하는 데이터 종속 문제가 존재함.
- 데이터베이스 관리 시스템은 응용 프로그램을 대신해서 데이터베이서에 접근하고 이를 관리하는 모든 책임을 지기 때문에 데이터베이스 구조가 변경되어도 응용 프로그램이 영향을 받지 않음. 즉 독립성이 확보됨.

3. 데이터를 동시 공유할 수 있음.

- 데이터베이스 관리 시스템은 데이터베이스에 통합된 데이터를 여러 응용 프로그램에서 공유하여 같은 데이터를 동시 접근할 수 있도록 지원함. 불필요한 중복을 제한할 수 있음.

4. 데이터 보안이 향상됨.

- 파일 시스템은 중복된 모든 파일의 보안을 같은 수준으로 유지하기 어려움.
- 데이터베이스 관리 시스템은 데이터베이스를 이용해 데이터를 중앙 집중식으로 관리하기 때문에 데이터에 대한 효율적인 접근 제어가 가능함.

5. 데이터 무결성을 유지할 수 있음.

- 데이터 무결성은 저장된 데이터 값의 정확성을 의미함.
- 데이터에 대한 연산이 수행될 때 마다 유효성을 검사하여 데이터 무결성을 유지할 수 있게 해줌.

6. 표준화 할 수 있음.

- 데이터에 대한 모든 접근이 데이터베이스 관리 시스템을 통해 이루어지기 때문에 데이터에 접근하는 방법, 데이터 형식과 구조 등을 표준화하기 쉬움.

7. 장애 발생시 회복인 가능함.

- 데이터베이스 관리시스템은 장애가 발생해도 데이터 일관성과 무결성을 유지하면서 데이터를 장애가 발생하기 이전 상태로 복구하는 회복 기능을 지원함.

8. 응용 프로그램 개발 비용이 줄어듬.

- 데이터에 대한 모든 관리를 응용 프로그램 대신 데이터베이스가 관리 시스템이 담당하기 때문에 파일 시스템을 사용할 때 보다 응용 프로그램 개발 비용이 적게듬.



#### 데이터베이스 관리 시스템의 단점

> 데이터베이스 관리 시스템은 장점이 많지만 모든 상황에 적합한 것은 아님.

1. 비용이 많이 든다.

- 파일 시스템은 운영체제와 함게 설치되므로 따로 구매비용이 들지 않음.
- 데이터베이스 관리 시스템은 따로 설치해야 하므로 구매 비용이 많이 듬.

2. 백업과 회복 방법이 복잡하다.

- 데이터베이스는 데이터양이 많아 구조가 복잡하고, 여러 사용자의 동시 공유를 지원하므로 장애가 발생했을 때 원인과 상태를 정확히 파악하기 어렵다.

3. 중앙집중 관리로 인한 취약점이 존재함.

- 모든 데이터가 데이터베이스에 통합되어있고 이에 대한 관리 책임이 데이터베이스 관리 시스템에 집중되어, 데이터베이스나 데이터베이스 관리 시스템에 장애가 발생하면 전체 시스템의 업무 처리가 중단됨.



## 4. 데이터베이스 관리 시스템의 발전 과정

> 사용하는 데이터 모델에 따라 네트워크 DBMS, 계층 DBMS, 관계 DBMS, 객체지향 DBMS, 객체관계 DBMS로 구분할 수 있음. 데이터베이스 관리 시스템은 발전 과정에 따라 다음 4세대로 나누어볼 수 있음.

#### 1세대 데이터베이스 관리 시스템 : 네트워크, 계층 DBMS

- 1960 ~ 1970년대에 사용됨.
- 네트워크 DBMS는 데이터베이스를 노드와 간선을 이용한 그래프 형태로 구성하는 네트워크 데이터 모델을 사용함. 데이터베이스의 구조가 복잡하고 변경하기 어렵다는 단점이 있음. 대표적인 예로는 1960년대 초에 개발된 IDS가 있음.
- 계층 DBMS는 데이터베이스를 트리 형태로 구성하는 계층 데이터 모델을 사용함. 네트워크 DBMS보다 구조가 단순하지만 복잡한 현실 세계의 모습을 부모 자식 관계가 명확한 트리형태로 표현하기 힘들고, 구조 변경이 어렵다는 단점이 있음. 대표적인 예로는 1960년대 후반에 개반된 IMS가 있음.



#### 2세대 데이터베이스 관리 시스템: 관계 DBMS

- 1980년 초반 ~
- 관계 DBMS는 데이터베이스를 테이블 형태로 구성하는 관계 데이터 모델을 사용함. 데이터베이스를 단순하고 이해하기 쉬운 구조로 구성한다는 장점이 있음. 대표적인 관계 DBMS로는 오라클, MYSQL, 액세스 등이 있음.



#### 3세대 데이터베이스 관리 시스템: 객체지향,객체관계 DBMS

- 1980년대 후반 ~ 
- 객체 지향 DBMS는 객체지향 프로그래밍에서 도입한 객체라는 개념을 이용해 데이터베이스를 구성하는 객체지향 데이터 모델을 사용함. 새로운 유형의 데이터를 저장하고, 데이터에 대한 복잡한 분석 및 처리를 지원함. 대표적인 예로는 오투, 온투스, 젬스톤 등이 잇음.
- 1990년대 후반 ~
- 객체관계 DBMS는 관계 데이터 모델에 객체지향 개념을 도입한 객체 관계 데이터 모델을 사용함. 쉽게 말해 객체 DBMS와 관계 DBMS의 개념을 통합한 것.



#### 4세대 데이터베이스 관리 시스템 : NoSQL, NewSQL DBMS

- 수많은 사람들이 소셜 네트워크 서비스를 폭발적으로 이용하면서 사진, 동영상, 검색 로그와 같은 비정형 데이터가 대량으로 생산되는 시대적 흐름에 따라 등장함.
- NoSQL은 관계 DBMS가 강점으로 제시하던 안정성과 일관성 유지를 위한 복잡한 기능을 포기하고, 데이터 구조를 미리 정해두지 않기 때문에 비정형 데이터를 저장하고 처리하는데 적합함. 확장성이 뛰어나 여러대의 서버 컴퓨터에 데이터를 분산하여 저장하고 처리하는 환경에서 주로 사용함. 대표적인 예로는 몽고디비, 오리엔트DB 등이 있음.
- 근데, 기존 기업들이 정형 데이터는 관계 DBMS로 처리해야하고, 비정형 데이터는 NoSQL를 도입해 처리해야하는 부담이 생김. 이를 배경으로 NewSQL DBMS가 등장함. 안정성과 일관성을 유지하면서도 SQL을 이용해 다양하고 복잡한 데이터 처리를 편리하게 요청 할 수 있음. 즉, 관계 DBMS의 장점 + NoSQL의 확장성 및 유연성을 모두 지원함. 대표적인 예로는 구글 스패너, 볼트DB, 누오DB등이 있음.
